# 🎵 音频URL频率响应功能使用指南

## 功能概述

这个新的频率响应模块允许您在加载不同的音乐URL时，根据音乐的**中高低频**特征动态调整动效。动效会根据音乐的频率成分实时变化，创建更加沉浸式和同步的视觉体验。

## 核心组件

### 1. **FrequencyAnalyzer.ts** - 频率分析模块
- 提供详细的频率波段分析（6个频段）
- 计算频率强度和平滑处理
- 支持自定义配置和灵敏度调整

**频率波段划分：**
```
低频段 (20-250Hz)：
  - 超低频 (20-60Hz)：影响整体动感
  - 低频 (60-250Hz)：强劲感和冲击力

中频段 (250-2000Hz)：
  - 低中音 (250-500Hz)：人声范围
  - 中音 (500-2000Hz)：乐器中间层

高频段 (2000-20000Hz)：
  - 高中音 (2000-6000Hz)：明亮感
  - 高频 (6000-20000Hz)：细节和空气感
```

### 2. **FrequencyVisualEngine.ts** - 视觉效果引擎
- 将频率数据转换为可视化参数
- 计算基于频率的缩放、旋转、颜色等效果
- 支持多种视觉增强效果

### 3. **FrequencyResponsePanel.tsx** - 控制面板
在右侧面板的 AudioUrlPanel 下方，提供以下控制：

#### 主要功能：
- **启用/禁用开关**：控制频率响应是否生效
- **实时频率显示**：彩色柱状图显示当前音乐的低中高频强度
- **低频反应** (20-250Hz)：调整低频对动效的影响程度 (0-3.0)
- **中频反应** (250-2000Hz)：调整中频对动效的影响程度 (0-3.0)
- **高频反应** (2000-20000Hz)：调整高频对动效的影响程度 (0-3.0)
- **频率平滑**：控制频率数据的平滑程度，越低越敏捷 (0.1-1.0)

#### URL音频专属参数：
- **URL平滑度**：音频流数据的平滑处理程度
- **节拍敏感度**：控制节拍检测的灵敏程度 (0.5-2.0)

## 使用步骤

### 1. 加载音乐URL
```
1. 打开右侧面板的 "AUDIO URL" 部分
2. 在输入框中粘贴音乐URL (需要支持CORS)
3. 点击 "CONNECT URL" 按钮
4. 等待音乐加载并开始播放
```

### 2. 调整频率响应
```
1. 向下滚动到 "FREQUENCY RESPONSE" 面板
2. 点击顶部的开关启用功能
3. 查看实时频率显示了解音乐特征
4. 根据需要调整各个频率段的反应强度
```

### 3. 优化效果
```
- 低频反应强化：用于低音重的音乐(电子、摇滚)
  推荐值：1.5-2.5
  
- 中频反应强化：用于人声为主的音乐(流行、说唱)
  推荐值：1.0-1.5
  
- 高频反应强化：用于高音多的音乐(古典、合成器)
  推荐值：1.2-2.0
  
- 频率平滑：
  - 低值(0.2-0.4)：更敏捷、反应更快
  - 高值(0.8-0.95)：更平稳、过渡更柔和
```

## 效果说明

### 频率如何影响动效？

1. **低频 (20-250Hz)**
   - 影响：粒子缩放、底层层次感、整体震感
   - 表现：画面的上下波动、粒子密度变化
   - 最佳用途：强调低音效果

2. **中频 (250-2000Hz)**
   - 影响：旋转速度、几何形状变化、中层细节
   - 表现：画面的旋转、形状扭曲
   - 最佳用途：跟随人声和主旋律

3. **高频 (2000-20000Hz)**
   - 影响：颜色变化、故障效果、高频闪烁
   - 表现：屏幕故障、色差分离、细节闪烁
   - 最佳用途：增加尖锐的视觉冲击

### 动效表现

根据启用的效果和频率反应设置，您会看到：

- **Bloom Intensity**：根据全频域强度动态调整
- **Glitch Amount**：根据高频强度变化，制造"数字故障"效果
- **Chromatic Aberration**：根据高频调整RGB分离程度
- **Scene Scaling**：根据频率组合动态调整场景大小
- **Rotation**：根据频率组合改变旋转速度和方向

## 技术细节

### 频率分析算法
- FFT大小：2048
- 采样率：44100Hz
- 频率分辨率：~21.5Hz/bin
- 平滑系数：可配置(0.1-1.0)

### 实时处理
- 无延迟处理
- 60fps+ 帧率
- GPU加速渲染
- 动态参数更新

### 兼容性
- 支持的URL：MP3、WAV、OGG、FLAC (需要CORS支持)
- 浏览器：Chrome、Firefox、Safari、Edge (现代版本)
- 麦克风备选：如果URL失败会自动切换到麦克风输入

## 故障排除

### 音乐不播放？
- 检查URL是否正确和可访问
- 确保URL所在的服务器支持CORS
- 尝试使用其他音乐URL测试

### 频率响应无效？
- 确保在 FrequencyResponsePanel 中启用了频率响应
- 检查是否使用的是 URL 音频源（不是麦克风或模拟）
- 调整频率平滑参数

### 效果太强/太弱？
- 调整各个频率段的反应强度
- 增加或降低频率平滑值
- 在场景中查看实时效果并实时调整

### 性能下降？
- 降低频率平滑值（会增加平滑度但可能需要更多计算）
- 关闭不需要的特效（FX面板）
- 使用"Performance Controls"中的速度控制

## 扩展功能

您可以进一步扩展此系统：

1. **自定义频率波段**
   修改 FrequencyAnalyzer.ts 中的 analyze() 方法

2. **新的视觉效果**
   在 FrequencyVisualEngine.ts 中添加新的计算方法

3. **频率预设库**
   在 useStore.ts 中添加预设配置

4. **实时频率可视化**
   创建一个独立的频率谱分析仪组件

## 示例代码

### 在自定义组件中使用频率数据
```typescript
import { audioEngine } from '@/lib/AudioEngine';
import { frequencyVisualEngine } from '@/lib/FrequencyVisualEngine';

function MyComponent() {
  // 获取当前频率数据
  const { subBass, bass, mid, treble } = audioEngine.current;
  
  // 获取视觉效果参数
  const effects = frequencyVisualEngine.computeEffects();
  
  return (
    <div>
      低频强度: {effects.lowFreqScale}
      高频强度: {effects.highFreqScale}
    </div>
  );
}
```
